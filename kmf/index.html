Models are no more restricted to modeling environements. The rise of Model@Runtime usages implies an efficient, multi-tenant and rich modeling framework. KMF aims at offering the effenciency and all the functionalities to edit, compose, synchronize models on JVM and JS virtual machines.

About Kevoree Modeling Framework

It's a framework to generate code API from a domain metamoel for JS and JVM platforms. Reusable for various usages such as the storage on server-side but also presentation in a simple browser. @Runtime, it offers comparison, merge and incremental synchronisation. In short KMF generate from a metamodel a distributed model environnement.

When you think about Cloud systems...

You think of hosting nodes and hosted softwares.

Everything begin with a metamodel

A cloud is composed by


A collection of nodes identified by an ID.

Each nodes contains softwares identified by their names.

Use any eclipse graphical or textual editor to produce an ecore metamodel.

Then KMF generates POJO *like* API

Java


//create a cloud
Cloud cloud = factory.createCloud();
//create a node
Node myAmazonEC2node = factory.createNode();
myAmazonEC2node.setID("EC2_0");
//add node to your cloud
cloud.addNode(myAmazonEC2node);
//create a software
Softwzre myNginx = factory.createSoftware();
myNginx.setID("SRV0");
//attach the software to your new node
myAmazonEC2node.addNode(myNginx);

Easily navigate acess elements using IDs

Java


//select your node by ID
Node ec2node = cloud.findNodeByID("EC2_0");
//or with XPath style
ec2node = cloud.findByPath("nodes[EC2_0]");
//then select a software
Software soft = ec2node.findSoftwareByName("SRV0");
//or by path
soft=cloud.findByPath("nodes[EC2_0]/softwares[SRV0]");
//print the path of any element
System.out.println(soft.path(););
//print on console : nodes[EC2_0]/softwares[SRV0]

Model elements selection using paths is way faster rather than iteration. It simpler but also way fatser because similarly to a database ID or NoSQL index, KMF generate behind your code hashtable to fast lookup elements.

Listen events while working on models

Java


class ModelSyncListener implements ModelTreeListener {
    @Override
    public void elementChanged(ModelEvent event){
	   switch(event.getType()){
	     case ActionType.ADD:
	       System.out.println("Add "+event);
	       break;
	     case ActionType.SET:
	       System.out.println("Set "+event);
               break;
	   }
    }
}
//Add as Tree (meaning global) listener
cloud.addModelTreeListener(new ModelSyncListener());
//Std listener only listen local event
//cloud.addModelListener(new ModelSyncListener());
Node node = factory.createNode();
node.setID("EC1");
//Trigger event ADD 
cloud.addNodes(node);
//Trigger event SET
node.setID("EC2_1");

Efficiency

KMF takes efficiency seriously.
With a generated, you can expect the best performance at runtime for your modeling operations.

Ease of Use and Portable

Unlike other modeling framework, KMF provides cross compilation for Java and JS platforms. Moreover it does'nt needs any library @Runtime and is easy-to-use for any OO developper by leveraging simple POJO api.

Distributed

Leveraging incremental synchronization mecanism, KMF aims at building distribution modeling environement. It now possible to embed and synchronize your tablet, phone with your home computer.
